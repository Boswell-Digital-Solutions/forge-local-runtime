# NeuronForge Local — Service Role

## Role

NeuronForge Local is the bounded local inference and candidate-production substrate for the Forge local runtime layer.

It exists to provide meaningful local AI capability under contract on realistic local hardware without becoming an authority surface.

## Owns

NeuronForge Local owns:

- local model routing and profile selection
- task-contract execution for local inference lanes
- candidate output production
- bounded local inference orchestration within explicit task contracts
- route-class posture for local inference paths
- truthful degraded-mode reporting for inference surfaces
- contract-shaped output discipline
- local inference readiness and availability reporting

## Explicitly does not own

NeuronForge Local does not own:

- final semantic authority
- final workflow authority
- application business truth
- hidden autonomous decision loops
- silent acceptance or promotion of candidates into truth
- broad execution authority across the runtime
- policy authority for side effects
- durable semantic memory authority by default

## Why the boundary exists

Local AI capability is useful only if it remains honest about what it is doing.

The architectural risk is that candidate-producing systems quietly become decision surfaces.
That drift is unacceptable in the Forge local runtime.

NeuronForge Local can produce, suggest, compare, rank, or transform under contract.
It cannot silently decide.

## Local baseline requirement

NeuronForge Local is part of the constitutional local baseline.

The local runtime must include meaningful local-first candidate production and bounded local inference capability even when cloud services are unavailable.

## Cloud augmentation line

Cloud intelligence services may augment:

- larger models
- broader orchestration
- deeper research
- extended memory surfaces
- heavier asynchronous workflows
- higher-cost inference paths

Cloud augmentation does not erase the need for local meaningful candidate production.

## Authority status

NeuronForge Local is a candidate-producing substrate, not an authority layer.

All outputs remain candidates unless explicitly promoted by an external authority surface.

## Route-class posture

NeuronForge Local should distinguish route classes clearly, such as:

- baseline local route
- preferred local route
- degraded local fallback
- unavailable route

This posture should remain explicit so consuming applications and review surfaces understand trust and capability level.

## Contract discipline

NeuronForge Local must prefer structured task contracts over vague prompt-only invocation posture where the runtime requires deterministic reviewability.

Outputs should preserve:

- task identity
- route identity
- model/profile identity where appropriate
- degraded truth
- confidence or quality posture where supported
- constraints applied during execution

## Privacy sensitivity

NeuronForge Local may process sensitive user content locally.

It must therefore follow privacy-preserving observability, minimized diagnostics, and bounded retention posture.
Prompt or content capture must not become default observability behavior.

## Readiness and degraded behavior

NeuronForge Local must report reduced-trust states truthfully, including where appropriate:

- degraded_pre_start
- degraded_in_flight
- degraded_fallback_equivalent
- degraded_fallback_limited
- stale
- unavailable_dependency_block
- partial_success

It must not present fallback behavior as equivalent unless equivalence is actually justified.

## Split-trigger relevance

A split away from NeuronForge Local is justified only when a concern develops materially distinct:

- authority boundaries
- hardware/runtime posture
- schema/control language
- observability constraints
- anti-drift value

## Interaction posture with other services

### With DF Local Foundation
NeuronForge Local may rely on bounded substrate support for persistence and readiness artifacts.
It does not transfer authority to DF Local Foundation.

### With Cortex
NeuronForge Local may consume bounded syntax-level packages or retrieval-preparation artifacts from Cortex.
It does not convert Cortex into semantic authority, and Cortex does not become inference authority.

### With FA Local
NeuronForge Local may be invoked through approved runtime paths where policy and capability posture allow it.
That does not make FA Local the semantic authority, and it does not make NeuronForge Local the execution governor.

## Governing rule

NeuronForge Local must remain a bounded, contract-disciplined, candidate-producing inference substrate.
If a proposal implies silent authority promotion, hidden planning, or autonomous side-effect drift, it should be rejected.
